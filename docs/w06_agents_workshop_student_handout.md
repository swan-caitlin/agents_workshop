# Agents workshop handout  

## What you will do today  
You will delegate a data analysis task to an agent, then you will check whether the output is correct and defensible. You will run it twice.

- Round 1 is intentionally under-specified  
- Round 2 adds clearer constraints and deliverables
- Round 3 is edits and improvements 

## Part 1 Setup  
1) Sign into GitHub.  
2) Fork the workshop repo.  
3) Open Codex and connect to your repo

## Part 2 Write your first `AGENTS.md`  
In the root of the repo, open `AGENTS.md`. Add content under these headings.

### Headings  
1) Role and identity  
2) Communication style  
3) Personality and boundaries  
4) Working preferences

### Prompts to help you write  
- What should the agent call you  
- How cautious should it be  
- Should it ask before making changes  
- What tone helps you supervise  
- What would annoy you/make you happy

## Part 3 Round 1 Naïve run  
1) Read `docs/project_card.md`.  
2) Ask the agent to analyse the dataset and generate a report.

Use a minimal instruction, for example  
- “Analyse the dataset described in `docs/project_card.md` and generate a report.” Or copy and paste or Reword. 

While it runs, write brief notes  
- What did you assume the agent would do  
- What did you fail to specify  
- Did you feel in control, and why?

## Part 4 Round 1 Report card  
Open the report and complete the Google Form report card.  
You must choose one  
- Accept  
- Accept with amendments  
- Reject and restart

Write a justification.

## Part 5 Peer audit  
Swap with someone near you.  
- Read their report  
- Check at least one thing independently  
- Submit a report card for their work

## Part 6 Improve your instructions  
Update `AGENTS.md` based on problems you noticed.

Then create `SPEC.md`.  
This file should say more clearly what you want, what you do not want, and how success will be checked.

### Include  
- Deliverables  
- Constraints  
- Success checks  
- Stop rules

## Part 7 Round 2 Rerun  
Start from scratch asking for a new analysis - ask the agent to follow `SPEC.md` and regenerate the report.

Then repeat  
- Your self report card  
- A quick peer audit if time
- 
## Debrief  
Be ready to answer  
- What changed between Round 1 and Round 2  
- What constraints mattered most?
- What did you trust too quickly?
- What did you verify, and how?
